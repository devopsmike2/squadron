# Getting Support

Thanks for using Squadron! Here is where to go depending on what you need.

## I have a question / want to discuss an idea

Use **[GitHub Discussions](https://github.com/devopsmike2/squadron/discussions)**.
Good for: "how do I…", design ideas, show-and-tell, and anything open-ended.

> Maintainers: enable **Settings → General → Features → Discussions** to turn
> this on.

## I found a bug

Open a **[bug report](https://github.com/devopsmike2/squadron/issues/new?template=bug_report.yml)**.
Please include your version (or `:latest`), how you ran it (Docker / compose /
binary), and the smallest repro you can manage. Sanitize any secrets in logs.

## I want a feature

Open a **[feature request](https://github.com/devopsmike2/squadron/issues/new?template=feature_request.yml)**
and describe the problem you are trying to solve, not just the solution.

## I found a security vulnerability

**Do not open a public issue.** Follow [SECURITY.md](SECURITY.md) to report it
privately.

## Documentation

- New here? Start with the [Quickstart](docs/quickstart.md) — one `docker run`,
  then open `http://localhost:8080/quickstart`.
- No cloud account? Try [Demo mode](docs/demo.md).
- The full docs index lives in [`docs/`](docs/) and the
  [README](README.md#documentation).

## Commercial support

Squadron's OSS core is free and self-hostable forever. Org-scale needs (SSO/RBAC,
HA, audit retention, SLAs) are covered by the planned commercial tier — see
[What's OSS vs Enterprise](docs/oss-vs-enterprise.md).
